# 11ty (Eleventy) + Tailwind Development Standards

You are an expert 11ty developer. You prioritize build speed, zero-bundle-size HTML, and the 11ty Data Cascade.

## 1. Project Architecture & Tooling
- **Framework:** 11ty (Eleventy). **Strictly no React/Vue/Hydration** unless requested.
- **CSS:** Tailwind CSS. Use utility classes in templates. Avoid @apply in CSS files.
- **Engine:** Prefer Nunjucks (.njk) for logic-heavy templates; Liquid (.liquid) for simple ones.
- **Images:** Always use `@11ty/eleventy-img` for responsive markup. Never use raw `<img>` tags for local assets.

## 2. The Data Cascade & Collections
- **Global Data:** Check `src/_data/` before hardcoding strings.
- **Directory Data:** Use `[directory-name].11tydata.js` for folder-specific logic (e.g., setting a common layout or tags).
- **Drafts:** Respect the `draft: true` front-matter key. Ensure collections filter out drafts in production:
  `eleventyConfig.addCollection("posts", col => col.getFilteredByTag("post").filter(p => !p.data.draft));`

## 3. Component Strategy (Shortcodes vs. Includes)
- Use **Includes** (`{% include "..." %}`) for static snippets (headers/footers).
- Use **Shortcodes** (`{% myComponent %}`) for dynamic, reusable UI elements that require arguments.
- Use **Paired Shortcodes** for wrappers (e.g., custom containers or markdown callouts).

## 4. Performance & Output
- **HTML:** Must be semantic. Ensure `alt` text is present in data structures.
- **Permalinks:** Use clean URLs (e.g., `/blog/my-post/` instead of `/blog/my-post.html`).
- **Scripts:** Use standard `<script type="module">` for vanilla JS. Load only on pages that need it using front-matter (e.g., `hasMap: true`).

## 5. Cursor Interaction Rules
- **Before creating a file:** Check `eleventy.config.js` to see if `input` is set to `src` or the root.
- **When suggesting a new page:** Provide the full Front Matter block including `title`, `layout`, and `date`.
- **When adding a plugin:** Remind the user to run `npm install [plugin]` and show the exact `eleventyConfig.addPlugin` snippet.
